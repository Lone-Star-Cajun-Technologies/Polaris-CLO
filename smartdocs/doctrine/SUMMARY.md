# Summary: doctrine

## Purpose
Canonical doctrine tree for SmartDocs.

## Key behaviors
- `active/` carries current doctrine.
- `candidate/` holds drafts under review.
- `deprecated/` is archival.
- High-authority doctrine promotion is review-gated; candidates should not be promoted to `active/` by automated ingest.

- **Parent**: `smartdocs`
