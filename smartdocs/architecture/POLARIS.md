# smartdocs/architecture

## Purpose

Architecture docs for SmartDocs live here. Use this folder for stable system structure, routing, and integration notes.

## What belongs here

- Architecture and design docs
- Cross-cutting system explanations
- Stable references that inform doctrine and specs
- The Quality Control lifecycle and operational boundaries note for POL-471 in
  [quality-control-lifecycle.md](quality-control-lifecycle.md)

## What does not belong here

- Raw ingest staging
- Generated runtime artifacts
- Ephemeral scratch notes

## Editing rules

- Prefer concise, durable architecture guidance.
- Keep links to doctrine/specs current.
